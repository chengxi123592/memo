## 打包虚拟环境

### 安装 conda-pack

```
conda install -c conda-forge conda-pack
```

### 打包环境

```
# Windows 上打包
conda pack -n yolo_new -o yolo_windows.zip

# Linux 上打包
conda pack -n yolo_new -o yolo_linux.tar.gz
```

```
# ../env/目录下创建文件夹
mkdir [文件夹名]

# 解压环境至文件夹
tar -zxvf data_process.tar.gz -C data_process/
```

## conda常用命令

### 创建虚拟环境

```
conda create --name jupyter_venv python=3.8
```

### 删除某个虚拟环境

```
conda remove -n your_env_name --all
```

### 复制某个虚拟环境

```
conda create --name new_env_name --clone old_env_name
```

### 分享/备份一个虚拟环境

- 一个分享环境的快速方法就是给他一个你的环境的`.yml`文件。

- 首先激活要分享的环境，在当前工作目录下生成一个`environment.yml`文件。

  ```
  conda env export > environment.yml
  ```

- 对方拿到`environment.yml`文件后，将该文件放在工作目录下，可以通过以下命令该文件创建环境即可。

  ```
  conda env create -f environment.yml
  ```

### 使用scp远程传输文件

- 从本地机器复制文件到远程机器

  ```
  scp /path/to/local/file username@remote_host:/path/to/remote/directory/
  # 示例
  scp /home/user/file.txt user@192.168.1.10:/home/user/
  ```

- 从远程机器复制文件到本地

  ```
  scp username@remote_host:/path/to/remote/file /path/to/local/directory/
  ```

- 复制整个目录

  ```
  # 本地到远程
  scp -r /path/to/local/directory username@remote_host:/path/to/remote/directory/
  # 远程到本地
  scp -r username@remote_host:/path/to/remote/directory/ /path/to/local/directory/
  ```

- 显示进度条

  ```
  scp -v /path/to/local/file username@remote_host:/path/to/remote/directory/
  ```

## Docker常用命令

### 查看镜像

```
docker images
```

### 删除镜像

```
docker rmi  镜像名/镜像ID #若镜像在运行则会报错
docker rmi -f 镜像名/镜像ID #强制删除一个
```

### 创建容器

```
docker run -d --name <新建容器名> -v <将当前目录挂载到容器目录> --gpus all <镜像名称> tail -f /dev/null 
# 示例
docker run -d --name mmpretrain_1 -v $(pwd):/workspace --gpus all mmpretrain:cuda12.8 tail -f /dev/null 
```

### 查看容器

```
docker ps
docker ps -a # 查看所有容器
```

### 启动容器

```
docker start <容器名>
docker restart <容器名>
```

### 停止容器

```
docker stop <容器名>
docker kill <容器名>    # 强制停止
```

### 进入容器

```
docker exec -it <容器名> bash
```

### 删除容器

```
#删除一个
docker rm <容器名>  
docker rm -f <容器名>  #强制删除
```

### 文件互传

```
# 主机复制到容器中
docker cp /host/path container_id:/container/path
docker cp $(pwd)/. mmpretrain_1:/workspace
# 容器复制主机中
docker cp container_id:/path/to/file /host/path
```

## Dlib库错误处理

- E1097 未知特性 “no_init_all“

  将winnt.h头文件中

  ```
  #if (_MSC_VER >= 1915)
  #pragma warning(disable:4845)   // __declspec(no_init_all) used but d1initall not set
  #endif
  ```

  这一预编译头之后加上

  ```
  #if (_MSC_VER >= 1915)
  #define no_init_all deprecated
  ```

  > [!NOTE]
  >
  > winnt.h只有管理员权限可以保存，需要右键属性-->安全-->编辑，将普通用户的权限改为可以写入

- USER_ERRORinconsistent_build_configurationsee_dlib_faq_1

  在dlib\test_for_odr_violations.h文件中，注释掉ENABLE_ASSERTS这一段

  ![image-20260124092722688](C:\Users\dell\AppData\Roaming\Typora\typora-user-images\image-20260124092722688.png)

- USER_ERRORinconsistent_build_configurationsee_dlib_faq_2

  在dlib/config.h，最后一行增加：

  ```
  #define DLIB__CMAKE_GENERATED_A_CONFIG_H_FILE
  ```

  之后重新编译或许就可以正常启动啦！

## ONNX Runtime库错误处理

- onnxruntime constexpr 函数 “Ort::Float16_t::Float16_t“ /“Ort::BFloat16_t::BFloat16_t“ 不会生成常数表达式问题

  ```
  修改ommxruntime_float16.h文件内容，将两处uint16 val{0} --> val
  ```


- 返回代码为: -1073741701
  - **依赖库位数不匹配（最常见）**：这是导致0xC000007B的头号原因。你的项目是使用 **x64** 模式编译的（从路径 `x64\Debug`可以看出）。但是，你链接或运行时加载的某个库（如 `onnxruntime.dll`， `opencv_worldxxx.dll`，或其他C++运行时库）是 **32位 (x86)** 版本的。混合32位和64位的DLL必然导致此错误。
  - **C++运行时库冲突**：你的项目设置（`/MDd`, `/MTd`等）与所使用库的编译设置不兼容。例如，你用 `/MTd`（静态链接运行时库）编译，却链接了一个用 `/MDd`（动态链接运行时库）编译的OpenCV或ONNX Runtime库。
  - **依赖的DLL文件缺失**：将以下必需的 **x64版本** 的DLL文件复制到你的可执行文件（`*.exe`）所在的目录（即 `x64\Debug`）：onnxruntime.dll（可能还有其他相关DLL，如 onnxruntime_providers_shared.dll等，取决于你下载的包）。

- 返回代码：-1073741515


  - **dll文件缺少：**检查项目所需的.dll文件是否确实，如DMS项目中依赖的 DLL 在这两个目录：

    - **OpenCV**: `...\Win\opencv-3.4.1-vc14_vc15\opencv\build\x64\vc14\bin\`
    - **ONNX Runtime**: `...\Win\onnxruntime-win-x64-gpu-1.18.0\lib\`

  - **配置运行环境：**如果IDE使用的Clion，需要运行配置中添加 PATH 环境变量


    - 点击CLion右上角运行配置下拉框 → **Edit Configurations...**
    
    - 选中你的 `00_C__` 配置
    
    - 点击 **Environment variables** 右边的文档图标按钮，会弹出环境变量编辑窗口，添加一条：
    
      ```
      Name>>PATH
      Value>>E:\Projects\20260226_Driver_Monitor_System\00_C++\Win\opencv-3.4.1-vc14_vc15\opencv\build\x64\vc14\bin;E:\Projects\20260226_Driver_Monitor_System\00_C++\Win\onnxruntime-win-x64-gpu-1.18.0\lib;%PATH%
      ```
    
      然后点 **OK** → **Apply** → **OK**，再运行程序。

- ONNX Runtime GPU 加载错误 **126**

  运行日志中出现：

  ```
  [ONNXRuntimeError] : 1 : FAIL : LoadLibrary failed with error 126 ""
  when trying to load "onnxruntime_providers_cuda.dll"
  ```

  或调试器输出：

  ```
  CUDAExecutionProvider 加载失败
  LoadLibrary failed with error 126 (找不到指定模块)
  ```

  问题通常出现在 **CUDA/cuDNN/路径依赖** 或 **DLL 加载冲突** 上。

  1. 使用 Dependencies 工具分析 DLL 依赖，打开onnxruntime_providers_cuda.dll查看是否缺少不要依赖，如果显示红色 ❌，说明 ONNX Runtime 的依赖版本与当前 CUDA 运行库不匹配。![image-20260309164649832](C:\Users\dell\AppData\Roaming\Typora\typora-user-images\image-20260309164649832.png)
  2. 检查路径冲突，确保程序运行目录下没有旧版的**onnxruntime.dll**、**onnxruntime_providers_cuda.dll**

  

## MMCV源码安装

```
# [1]创建虚拟环境
conda create --name mm python=3.10
# [2]激活环境
activate mm
# [3]安装torch CUDA 12.8
pip3 install torch torchvision --index-url https://download.pytorch.org/whl/cu128
# [4] 设置计算能力和查看
export TORCH_CUDA_ARCH_LIST=12.0
echo $TORCH_CUDA_ARCH_LIST
# [5] 设置和查看MMCV_WITH_OPS
export MMCV_WITH_OPS=1
echo $MMCV_WITH_OPS
# [6]设置和查看MAX_JOBS
export MAX_JOBS=8
echo $MAX_JOBS
# [7] mmcv源码安装
将mmcv源码移至site-packages文件夹内解压缩
python setup.py develop 
```
