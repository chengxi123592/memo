## 连扳调试

RKNN Toolkit2的连板功能一般需要更新板端的 rknn_server 和 librknnrt.so/librknnmrt.so，并且手动启动 rknn_server 才能正常工作。
rknn_server: 是一个运行在板子上的后台代理服务，用于接收PC通过USB传输过来的协议，然后执行板端runtime对应的接口，并返回结果给PC。

- 服务器端安装adb

  ```
  sudo apt-get install adb
  ```

- ADB调试配置：（在切换ADB模式之前，请断开USB口的外部供电）

  ```
  # 板端输入
  cd /usr/local/cookoo/examples/
  sudo ./example-cookoo_pin SET_DEVICE 1
  
  # 服务器端查看设备是否连接
  adb devices
  ```

- librknnrt.so: 是一个板端的runtime库；

- 以RK3588 Linux平台为例，进入到rknn-toolkit2-master/rknpu2/runtime/RK3588/目录；

  - BOARD_ARCH在64位Linux系统中，对应aarch64目录，在32位系统，对应armhf目录

    ```
    adb push Linux/rknn_server/${BOARD_ARCH}/usr/bin/下的所有文件到/usr/bin目录
    adb push Linux/librknn_api/${BOARD_ARCH}/librknnrt.so到/usr/lib目录
    ```

  - 进入板子的**串口**终端，执行：

    ```
    chmod +x /usr/bin/rknn_server
    
    killall rknn_server
    nohup /usr/bin/rknn_server > /dev/null&
    ```

  - 串口查看rknn_server详细日志

    ```
    # 进入板端,设置日志等级
    export RKNN_SERVER_LOGLEVEL=5
    # 重启rknn_server进程(若固件没有自启动rknn_server)
    nohup /usr/bin/rknn_server > /dev/null&
    ```

  - 查看进程

    ```
    ps -ef|grep rknn_server
    ```

- **如果不使用连板推理功能**，只需要通过shell连接将Linux/librknn_api/${BOARD_ARCH}/librknnrt.so存到板子的/usr/lib目录中即可（若没有root权限，可将文件存于用户目录下，使用sudo cp librknnrt.so /usr/lib拷贝过去）

## 量化精度分析

可输出32浮点模型权重（.pt .onnx等）与rknn权重在推理同一组图像时的误差精度分析报告error_analysis.txt，评估量化精度是否满足要求。

```
model_path = "yolov5_7.0_best.onnx"
# 使用RKNN方法创建RKNN对象
rknn = RKNN(verbose=True)    

# 使用config接口配置要生成的RKNN对象 
rknn.config(
    mean_values=[123.675, 116.28,103.53], 
    std_values=[58.395, 58.395, 58.395],
    target_platform='rk3588'
)

# 使用load_xxx接口加载常用深度学习模型
rknn.load_pytorch(
     model=model_path,
     input_size_list=[[1, 3, 640, 640]]
)

rknn.load_onnx(
    model=model_path,
    input_size_list=[[1, 3, 640, 640]]
)

# 使用build接口构建RKNN模型 
rknn.build(
    do_quantization=True,
    dataset='dataset.txt'
)

# 使用accuracy_analysis 接口进行模型量化精度分析 
rknn.accuracy_analysis(
    inputs=["images/1.jpg"], # inputs用来表示进行推理的图像
    output_dir="analysis/",  # output_dir表示精度分析的输出目录
    target="rk3588",         # target表示目标硬件平台
    device_id=None,          # device_id表示设备的编号
)

# 使用release接口释放RKNN模型
rknn.release()
```

分析报告error_analysis.txt中包含下列关键参数：

- PC端模拟器误差（simulator_error）：评估模拟器的输出与32浮点模型之间的差异：
  - entire（整体累积误差）：比较模拟器逐层输出与32浮点模型的差异，误差会逐层累积。这反映了模拟器在多层级联时的整体精度。
  - single（单层误差）：单独比较每个层的输出差异。这能更纯粹地反映模拟器单个计算层的实现精度，避免了前面层误差的传播。
  - cos（余弦距离）：值越大误差越小
  - euc（欧式距离）：值越小误差越小
- RKNN-NPU误差（runtime_error）：评估RKNN-NPU实际推理与32浮点模型之间的差异

## 混合量化调优

可输出量化配置文件xxx.quantization.cfg，对每层量化自定义优化。将优化后的配置文件重新导入并据此进行量化。

```
model_path = "yolov5_7.0_best.onnx"
# 使用RKNN方法创建RKNN对象
rknn = RKNN(verbose=True)    

# 使用config接口配置要生成的RKNN对象 
rknn.config(
    mean_values=[123.675, 116.28,103.53], 
    std_values=[58.395, 58.395, 58.395],
    target_platform='rk3588'
)

# 使用load_xxx接口加载常用深度学习模型
rknn.load_pytorch(
     model=model_path,
     input_size_list=[[1, 3, 640, 640]]
)

rknn.load_onnx(
    model=model_path,
    input_size_list=[[1, 3, 640, 640]]
)

# 使用hybrid_quantization_step1接口进行混合量化第一步
rknn.hybrid_quantization_step1(
    dataset="dataset.txt",  # 表示混合量化所需要的数据集
    rknn_batch_size=None,   # 自动调整模型输入batch数量
    proposal=True,  # 设置为True，可以自动产生混合量化的配置建议值
    proposal_dataset_size=5 # 量化数据集规模         
)

# 使用hybrid_quantization_step2接口进行混合量化第二步
rknn.hybrid_quantization_step2(
    model_input='xxx.model',  # 第一步生成的模型文件
    data_input='xxx.data',    # 第一步生成的配置文件
    model_quantization_cfg='xxx.quantization.cfg' # 第一步生成的量化配置文件
)

# 使用accuracy_analysis接口对调优后的量化进行量化精度分析 
rknn.accuracy_analysis(
    inputs=["images/1.jpg"],  
    output_dir="quantization/",         
    target="rk3588",               
    device_id=None,                
)

# 使用expoer_rknn接口导出RKNN模型(可导出也可不导出) 
rknn.export_rknn(
    export_path=export_path
)
# 使用release接口释放RKNN模型
rknn.release()

```

量化配置文件xxx.quantization.cfg中包含下列关键参数：

- 自定义量化配置（custom_quantize_layers）：指定了哪些层的输出不进行低精度量化

## 常见问题

### 问题1 5037端口监听

Debian系统上rknn_server服务已经后台启动, 但是连板推理时依旧有如下报错

```
D NPUTransfer: ERROR: socket read fd = 4, n = -1: Connection reset by peer
D NPUTransfer: Transfer client closed, fd = 4
E RKNNAPI: rknn_init,  server connect fail!  ret = -9(ERROR_PIPE)!
E build_graph: The rknn_server on the concected device is abnormal, please start the rknn_server on the device according to:
               https://github.com/airockchip/rknn-toolkit2/blob/master/doc/rknn_server_proxy.md
```

解决方法

这通常是由于Debian固件上的adbd程序没有监听5037端口导致的，可以在板子上执行以下命令来判断:

```
netstat -n -t -u -a
或
netstat -tulnp | grep 5037
```

如果输出结果中没有5037端口,则在服务器端执行下列命令下载和更新adbd程序, 并重启板子;否则,跳过下列步骤。

```
wget -O adbd.zip
https://ftzr.zbox.filez.com/v2/delivery/data/7f0ac30dfa474892841fcb2cd29ad924/adbd.zip

unzip adbd.zip

adb push adbd/linux-aarch64/adbd /usr/bin/adbd
```

进入设备shell命令，增加adbd的可执行权限

```
adb shell "chmod +x /usr/bin/adbd"
adb reboot
```

重启设备后，按照启动步骤启动rknn_server服务，再次尝试连板推理。

### 问题2 accuracy_analysis 接口

使用accuracy_analysis python接口使用时，因为模型太大，板端存储容量不够导致运行失败。

解决方法

将/userdata/目录作为精度分析时保存中间结果文件的目录，再重启rknn_server，PC端执行命令如下：

```
# 创建dumps文件夹
adb shell "mkdir -p /userdata/dumps"
# 将/userdata/目录作为精度分析时保存中间结果文件的目录，再重启rknn_server
adb shell "killall rknn_server"
adb shell "export RKNN_DUMP_DIR=/userdata/dumps && nohup /usr/bin/rknn_server >/dev/null"&
```
